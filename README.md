# Machine Learning-Driven Fuel Optimization for Power Generation Plants

---

## 1. Project Description and Objectives

This project aims to optimize fuel usage and minimize fuel loss at telecom base stations located in regions with unreliable national grid access. Exploratory data analysis identifies key factors contributing to fuel inefficiency. Four supervised learning models Gradient Boosting, Random Forest, Neural Network, and Lasso are implemented to forecast fuel consumption. Among these, the Gradient Boosting model achieves the highest Nash efficiency value of 99.1%, indicating superior predictive performance.

### Key Findings

- **Running Time** and **Fuel per Period** are the two strongest predictors (identified via ExtraTreesRegressor feature importance).
- **Gradient Boosting** (after hyperparameter tuning) achieves the highest Nash-Sutcliffe Efficiency (NSE = 0.992), outperforming all other models.
- All four tuned models achieve NSE > 0.91, confirming the practical viability of the proposed approach.The analysis reveals a potential fuel savings of approximately 84,000 liters.

## 2. Environment Setup and Hardware Requirements

### Hardware

| Resource | Requirement |
|---|---|
| CPU | Any modern multi-core CPU (no GPU required) |
| RAM | ≥ 8 GB recommended (dataset is ~18 000 rows) |
| Disk | ~500 MB for raw data, models, and figures |

All computations use scikit-learn (CPU only). No CUDA or GPU libraries are required.

### Python Version

Python **3.7** is required. The codebase uses APIs that were deprecated or removed in later versions (e.g., `loss='ls'` in `GradientBoostingRegressor`, `sheetname=` in `pd.read_excel`, `sns.distplot`).

### Creating the Environment

```bash
# (Recommended) Create an isolated virtual environment
python3.7 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# Install all pinned dependencies
pip install -r requirements.txt
```

The key version pins and their rationale are documented inside `requirements.txt`:

| Package | Pinned version | Reason |
|---|---|---|
| scikit-learn | 0.21.3 | `loss='ls'` (GradientBoosting) removed in 1.0 |
| xlrd | 1.2.0 | `.xlsx` support dropped in xlrd 2.0 |
| seaborn | 0.9.0 | `sns.distplot` removed in 0.12 |
| yellowbrick | 0.9.1 | API in notebook targets this release |

## 3. Data Provenance and Acquisition

### Source

The raw dataset consists of a proprietary Excel workbook that describes part of the routine maintenance at telecom base stations:
```
Full_Data_Gen_only.xlsx   (sheet: Generator_only)
```

It contains 11 months of field-visit records (approximately 18 000 rows) for generator sites in the northern regions of Cameroon, capturing operational measurements such as hour-meter readings, fuel stock levels, running time, and site metadata.

**The raw file is NOT tracked in this repository** (`.gitignore` excludes `data/raw/`). Obtain the file through the appropriate internal data request process and place it at:

```
data/raw/Full_Data_Gen_only.xlsx
```

The processed CSV (`data/processed/cleaned_generator_data.csv`) is also excluded from version control and is regenerated by running the data pipeline (see Section 5).

## 5. Execution — How to Run the Pipeline

All commands are run from the **project root** directory.

### Step 1 — Data Preprocessing

Loads `data/raw/Full_Data_Gen_only.xlsx`, applies cleaning rules (outlier removal, column drops, cluster normalisation, feature engineering), and writes `data/processed/cleaned_generator_data.csv`.

```bash
python -m src.data.make_dataset
```

### Step 2 — Pre-training EDA Figures *(optional)*

Generate all exploratory figures into `reports/figures/EDA-figures/`.

```bash
# All EDA figures at once
python -m src.visualization.visualize pre all

# Or individually
python -m src.visualization.visualize pre distribution
python -m src.visualization.visualize pre feature_importance
python -m src.visualization.visualize pre correlation
python -m src.visualization.visualize pre boxplot
```

| Figure name | Description |
|---|---|
| `distribution` | KDE + probability plot of Fuel per Period |
| `feature_importance` | ExtraTrees feature importances (top 19) |
| `rate` | Consumption vs Consumption Rate scatter |
| `running_time` | Running Time vs Consumption |
| `correlation` | Numeric feature correlation heatmap |
| `boxplot` | Fuel consumption boxplot by geographic cluster |
| `scatter` | Scatter matrix of key numeric columns |

### Step 3 — Model Training

Loads the processed CSV, builds the feature matrix, runs 10-fold cross-validation, and serialises the four fitted models to `models/`. Also saves evaluation CSVs to `reports/evaluation-metrics/`.

```bash
python -m src.models.train_model
```

To run full hyperparameter tuning before training (slow — ~hours):

```bash
# Edit config.py and set tune=True, or call run_training(tune=True) directly
python -m src.models.train_model  # tune=True must be set in the __main__ block
```

### Step 4 — Post-training Diagnostic Figures *(optional)*

Requires fitted `.pkl` files in `models/`. Figures are saved to `reports/figures/training-figures/`.

```bash
# All post-training figures for all models
python -m src.visualization.visualize post all

# Or individually
python -m src.visualization.visualize post pred_vs_obs
python -m src.visualization.visualize post residuals
python -m src.visualization.visualize post learning_curve
python -m src.visualization.visualize post nse_boxplot
python -m src.visualization.visualize post r2_boxplot
```

| Figure name | Description |
|---|---|
| `pred_vs_obs` | Line overlay + PredictionError plot per model |
| `residuals` | Yellowbrick ResidualsPlot per model |
| `learning_curve` | Training vs validation RMSE vs training-set size |
| `nse_boxplot` | NSE distribution across 10 folds for all models |
| `r2_boxplot` | R² distribution across 10 folds for all models |

## 6. Results and Evaluation Metrics

All metrics are computed using **10-fold cross-validation** on the full dataset, with the best hyperparameters found via `RandomizedSearchCV`. The table below reflects the final results from the original study notebook.

### Model Performance Summary (after hyperparameter tuning)

| Model | NSE ↑ | RSR ↓ | RMSE (L) ↓ | Bias (L) | PBIAS (%) |
|---|---|---|---|---|---|
| **Gradient Boosting** | **0.9919** | **0.0819** | **12.77** | -0.20 | -0.09 |
| Random Forest | 0.9867 | 0.1086 | 17.12 | 0.97 | 0.39 |
| MLP Regressor | 0.9614 | 0.1889 | 29.74 | -0.19 | — |
| Lasso Regression | 0.9175 | 0.2853 | — | 0.07 | 0.02 |

- **NSE** (Nash-Sutcliffe Efficiency): 1 is perfect; values > 0.75 indicate very good model performance per USDA guidelines.
- **RSR** (RMSE-to-observations standard deviation ratio): < 0.50 is "very good"; all models qualify.
- **PBIAS**: percentage bias; values near zero indicate minimal systematic over/under-prediction.

The full per-fold NSE values and all six metrics per model are written to:
- `reports/evaluation-metrics/models-nse.csv`
- `reports/evaluation-metrics/models-statistic-metrics.csv`
